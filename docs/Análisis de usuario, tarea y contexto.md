Los usuarios objetivo del sistema son principalmente empleados y encargados de una ferretería. El empleado necesita consultar productos, revisar stock disponible y registrar movimientos cotidianos, como entradas o salidas de mercadería. El encargado, además de esas tareas, debe poder registrar nuevos productos, controlar alertas de stock bajo, revisar el estado general del inventario y cargar archivos Excel enviados por proveedores.

Las tareas principales del sistema se relacionan con la gestión operativa del stock. Entre ellas se encuentran consultar el inventario, buscar productos, registrar movimientos de entrada y salida, identificar productos con stock bajo e importar datos desde archivos Excel. Estas tareas son importantes porque permiten mantener actualizada la información del negocio y evitar problemas como faltantes de productos o inconsistencias en las cantidades disponibles.

El contexto de uso es un entorno laboral de ferretería, donde el usuario puede necesitar realizar consultas rápidas mientras atiende clientes o controla mercadería. Por este motivo, la interfaz debe priorizar la claridad, la velocidad de acceso a la información y la reducción de errores. Es probable que el sistema se utilice desde una computadora de escritorio o notebook dentro del local, aunque también podría adaptarse en el futuro a tablets o dispositivos móviles.

Una restricción importante es que los usuarios pueden no tener conocimientos técnicos avanzados. Por eso, el diseño debe evitar pantallas sobrecargadas, términos ambiguos o procesos largos. Las acciones principales deben estar visibles, los formularios deben ser simples y los mensajes del sistema deben indicar claramente si una operación fue exitosa o si ocurrió un error.
